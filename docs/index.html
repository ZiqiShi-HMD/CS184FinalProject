Summary

In this project we plan to create a set of shaders for the game Minecraft that will introduce complex rendering techniques and visuals into the game. Shaders of this variety already exist, however, we wish to learn more about OpenGL and GLSL programming by creating our own.

Problem Description

Our project doesn't necesarily aim to solve a "problem" per se, but the basic idea is that we want to improve the visual look of the game Minecraft. The default rendering for the game is quite boring (at least in our opinion), and adding effects such as shadow mapping or normal mapping on water would really add to the experience of playing the game.

Goals and Deliverables

Since we are making a shader pack for the game, we plan on having a "demo" where we make the shaders availaibe (on a public github repo prehaps) and allow other students to download them and run them locally on their machine. This will serve as an interactive demo allowing anyone who wants to try out our shaders on their own computer at home. However, we recognize that not everyone owns a copy of Minecraft or wants to spend time installing shaders, so we will also be taking screenshots of thew game running on our own comptuers with the shaders off and on, showing them side by side.

Since our project consists of altering the rendering code of a game, there is no way to objevtively measure how "good" our shaders are, since that's a matter of personal opinion. However, we plan on attempting to measure the quality of our implementation by comparing the look of our shader to the reference implementations that we will be using. For example, when we impelement our "improved water" with normal mapping and wave simulations, we will reference the ThinMatrix Youtube tutorial on the topic to make sure our implemention kinda looks like his. Of course, since we are simulating real world phenomena, we can also just use our eyes to see if our new water looks any more realistic or not.

Plan to Deliver:
  1. Impelement a simple cell shader for minecraft: This theoretically should be a very very simple shaderpack and will serve as a good "practice run" for the rest of the project.
  2. Add a "wind" effect to tree leaves to make them sway in the wind: This should be another simple shader, but this time it fits with the theme of the other effects, so it will be included in the final shaderpack.
  3. Add a dynamic shadow system and cast a shadow from the position of the sun: essentially, we want to add shadows using shadow mapping and cast a shadow on the terrain based on the direction of the sun. Adding shadows from light sources other than the sun should be much harder, from our understading.
  4. Significantly improve the look of water in the game: By default, minecraft's water looks like a bland-blue-nothingness. We can do better. We want to add a more realistic water rendering system that includes effects like tiny ripples in the water (using normal mapping), waves (in the vertex shader), implement a very basic version of the Fresnel effect to make the transparency of the water dependant on the angle at which one is looking.
Hope to Deliver:
  1. Water reflections and refractions: To really polish our water implementation, we would like to also add water reflections and refraction. We figure using some of the techniques we learned during shadow mapping should help us, but we are far less confident about this task, so we are putting it in the "hope" section.
  2. Add bloom to light sources: We have no idea how we would do this yet, but we think it would be really cool to add a bloom effect into the game. From our understading, this is just a post processing effect, but we are very unsure how we would integrate it into the game yet.
  3. Motion Blur: We also think it would be cool to add motion blur. We don't know anything about it's implementation, so we have no idea how hard this will actually be. I guess we'll find out...

  • A tutorial for implementing Cell shading in OpenGL
  • A series of tutorials on the basics of custom Minecraft shaders
